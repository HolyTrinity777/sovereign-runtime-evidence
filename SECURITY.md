# Security Policy

## Supported versions

This repository publishes a public evidence package for the Sovereign Runtime.

Supported public releases are documented in the repository tags and release notes. Security guidance applies to the current public release and any actively maintained public evidence package versions.

## Reporting a vulnerability

If you believe you have found a security vulnerability in this repository, please report it privately and do not open a public issue.

Preferred reporting channels:
- GitHub Security Advisory, if enabled for this repository.
- Private email to the project security contact: kaiven2342025@proton.me

Please include as much detail as possible, such as:
- A description of the issue.
- The files or components involved.
- Steps to reproduce the issue.
- Any relevant logs, screenshots, or proof-of-concept material.
- An assessment of the potential impact.


## Responsible disclosure

To protect users and maintainers, we ask that you:
- Do not disclose vulnerabilities publicly until they have been reviewed and addressed.
- Do not exploit the issue beyond what is necessary to demonstrate it.
- Do not share exploit details with third parties before coordination is complete.
- Allow a reasonable time for investigation and remediation.

## Response expectations

Upon receipt of a valid report, we aim to:
- Acknowledge the report promptly.
- Assess severity and scope.
- Investigate affected versions and artifacts.
- Develop and validate a fix if needed.
- Coordinate disclosure once remediation is available.

## Scope

This policy applies to the public repository artifacts, documentation, and any public shell components included in the evidence package.

It does not govern the private sovereign runtime core, which is maintained separately under NDA or commercial agreement.

## Safe reporting note

If your report concerns dangerous or sensitive payload material, please provide only the minimum necessary information for review and keep disclosure private until the issue is resolved.

## Thank you

We appreciate responsible security research and constructive reports that help improve the safety and integrity of this project.
